# tri-risk decision logic

## Что на входе

На входе факты и признаки процесса.

Примеры:
- есть документ или нет
- есть печать или нет
- есть подпись или нет
- срок подходит или нет
- есть задолженность или нет
- каких то данных не хватает

## Что на выходе

На выходе одно из 3 состояний:

- `1` = ok
- `0` = stop
- `M` = unknown

## Что это значит

### `1`
Данных хватает  
Условия выполняются  
Можно идти дальше

### `0`
Данных хватает  
Но условие не проходит  
Значит стоп или отказ

### `M`
Данных недостаточно  
И нельзя честно принять решение  
Нужно дособрать факты или документы

## Зачем это нужно

Обычная бинарная логика заставляет делать вид что решение уже можно принять

В реальных процессах это часто не так

Особенно в:
- страховании
- скоринге
- антифроде
- проверке заявок
- проверке документов
- enterprise decision systems

Поэтому tri-risk сохраняет отдельное состояние `M`
и позволяет не ломать процесс forced yes/no

## Пример 1

Возврат страховой премии

На входе:
- есть договор
- попадает ли случай в срок
- есть заявление
- есть паспорт
- есть ли долг

На выходе:
- `1` если все ок
- `0` если условие не проходит
- `M` если не хватает данных

## Пример 2

Проверка медицинского документа

На входе:
- есть печать
- есть подпись врача
- есть дата
- документ читаемый

На выходе:
- `1` если документ валиден
- `0` если есть явное несоответствие
- `M` если данных недостаточно

## Суть

tri-risk это не магия и не "ИИ сам решил"

Это decision logic для процессов где кроме "да" и "нет"
есть еще честное состояние

**данных пока недостаточно**
